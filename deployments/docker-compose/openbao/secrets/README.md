# OpenBao Initialization Secrets

This directory contains sensitive files generated during the OpenBao initialization process.

## ⚠️ CRITICAL SECURITY WARNING

The files in this directory contain **Root Tokens** and **Unseal Keys**. Anyone with access to these files can gain full administrative control over your OpenBao instance and all registered credentials.

- **NEVER** commit these files to version control.
- **NEVER** share these files over insecure channels.
- Restrict file permissions (e.g., `chmod 600`).

## Files

### [openbao-init.json](file:///home/ubuntu/dev/cloud-barista/mc-terrarium/deployments/docker-compose/openbao/secrets/openbao-init.json)

- **Generated by**: `openbao-init.sh` (called automatically during `make up` for the first time).
- **Contains**:
  - `root_token`: Used for administrative access and credential registration.
  - `unseal_keys_b64`: Shared keys required to unseal the vault after a restart.
- **Usage**: The `openbao-unseal.sh` and `openbao-register-creds.sh` scripts read this file automatically.

### [.gitkeep](file:///home/ubuntu/dev/cloud-barista/mc-terrarium/deployments/docker-compose/openbao/secrets/.gitkeep)

- A placeholder file to ensure this directory exists in the repository. It is the only file in this directory that should be tracked by Git.

## Cleanup

Running `make clean-all` will delete the `openbao-init.json` file. This is useful for a full reset, but requires you to run `make init` again to re-register credentials.
